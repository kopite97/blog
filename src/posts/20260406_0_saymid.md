## 가운데를 말해요

[가운데를 말해요](https://www.acmicpc.net/problem/1655)

### 문제

백준이는 동생에게 "가운데를 말해요" 게임을 가르쳐주고 있다. 백준이가 정수를 하나씩 외칠때마다 동생은 지금까지 백준이가 말한 수 중에서 중간값을 말해야 한다. 만약, 그동안 백준이가 외친 수의 개수가 짝수개라면 중간에 있는 두 수 중에서 작은 수를 말해야 한다.

예를 들어 백준이가 동생에게 1, 5, 2, 10, -99, 7, 5를 순서대로 외쳤다고 하면, 동생은 1, 1, 2, 2, 2, 2, 5를 차례대로 말해야 한다. 백준이가 외치는 수가 주어졌을 때, 동생이 말해야 하는 수를 구하는 프로그램을 작성하시오.

### 입력

첫째 줄에는 백준이가 외치는 정수의 개수 N이 주어진다. N은 1보다 크거나 같고, 100,000보다 작거나 같은 자연수이다. 그 다음 N줄에 걸쳐서 백준이가 외치는 정수가 차례대로 주어진다. 정수는 -10,000보다 크거나 같고, 10,000보다 작거나 같다.

### 출력

한 줄에 하나씩 N줄에 걸쳐 백준이의 동생이 말해야 하는 수를 순서대로 출력한다.

---

### 풀이

```java
import java.io.*;
import java.util.*;

public class Main {
    static PriorityQueue<Integer> MinPq;
    static PriorityQueue<Integer> MaxPq;
    public static void main(String[] args) throws IOException {

        BufferedReader br = new BufferedReader(new InputStreamReader(System.in));
        int n = Integer.parseInt(br.readLine());

        MinPq = new PriorityQueue<>();
        MaxPq = new PriorityQueue<>(Collections.reverseOrder());

        StringBuilder sb = new StringBuilder();
        for (int i = 0; i < n; i++) {
            int num = Integer.parseInt(br.readLine());
            int minSize = MinPq.size();
            int maxSize = MaxPq.size();

            if (minSize >= maxSize) {
                MaxPq.add(num);
            } else {
                MinPq.add(num);
            }

            int currentMid = 0;
            int currentMin = 0;


            if(!MaxPq.isEmpty() && !MinPq.isEmpty()){
                currentMid = MaxPq.peek();
                currentMin = MinPq.peek();
                if(currentMid > currentMin){
                    int swapFakeMid = MaxPq.poll();
                    int sawpRealMid = MinPq.poll();

                    MaxPq.add(sawpRealMid);
                    MinPq.add(swapFakeMid);
                }
            }

            sb.append(MaxPq.peek()).append("\n");
        }

        System.out.println(sb);

    }
}
```

데이터가 계속 추가되는 상황에서 중간값을 찾아야 하는 문제. 처음에는 단순하게 풀었다가 메모리 초과가 나서 어떻게 하면 메모리가 초과되지 않을지 고민했던 문제. 힙의 특성을 아주 제대로 이용한 문제다.  
최대 힙과 최소 힙을 두 개 선언하여 데이터의 절반씩 나누어 담는 발상만 하면 생각보다 금방 풀 수 있는 문제. 골드2인 이유는 그 발상이 떠오르기 쉽지 않은 이유라 생각한다. 사실 나도 힌트를 보지 않았으면 풀지 못했을 것..
