## 줄 세우기

[줄 세우기](https://www.acmicpc.net/problem/2252)

### 문제

N명의 학생들을 키 순서대로 줄을 세우려고 한다. 각 학생의 키를 직접 재서 정렬하면 간단하겠지만, 마땅한 방법이 없어서 두 학생의 키를 비교하는 방법을 사용하기로 하였다. 그나마도 모든 학생들을 다 비교해 본 것이 아니고, 일부 학생들의 키만을 비교해 보았다.

일부 학생들의 키를 비교한 결과가 주어졌을 때, 줄을 세우는 프로그램을 작성하시오.

### 입력

첫째 줄에 N(1 ≤ N ≤ 32,000), M(1 ≤ M ≤ 100,000)이 주어진다. M은 키를 비교한 횟수이다. 다음 M개의 줄에는 키를 비교한 두 학생의 번호 A, B가 주어진다. 이는 학생 A가 학생 B의 앞에 서야 한다는 의미이다.

학생들의 번호는 1번부터 N번이다.

### 출력

첫째 줄에 학생들을 앞에서부터 줄을 세운 결과를 출력한다. 답이 여러 가지인 경우에는 아무거나 출력한다.

---

### 풀이

```java
import java.util.*;
import java.io.*;

public class Main {
    static ArrayList<ArrayList<Integer>> graph;
    static int[] inDegree;
    public static void main(String[] args) throws IOException {

        BufferedReader br = new BufferedReader(new InputStreamReader(System.in));

        StringTokenizer st  = new StringTokenizer(br.readLine());
        int n = Integer.parseInt(st.nextToken());
        int m = Integer.parseInt(st.nextToken());

        graph = new ArrayList<>();
        inDegree = new int[n+1];
        for (int i = 0; i <= n; i++) {
            graph.add(new ArrayList<>());
            inDegree[i] = 0;
        }


        for (int i = 0; i < m; i++) {
            st = new StringTokenizer(br.readLine());
            int a = Integer.parseInt(st.nextToken());
            int b = Integer.parseInt(st.nextToken());

            graph.get(a).add(b);
            inDegree[b]++;
        }

        Queue<Integer> q = new LinkedList<>();
        for (int i = 1; i <= n; i++) {
            if(inDegree[i] == 0){
                q.offer(i);
            }
        }

        StringBuilder sb = new StringBuilder();

        while (!q.isEmpty()) {
            int current = q.poll();
            sb.append(current).append(" ");
            for(int next : graph.get(current)) {
                inDegree[next]--;
                if(inDegree[next] == 0){
                    q.offer(next);
                }
            }
        }

        System.out.println(sb);

    }
}
```

요즘 따라 그래프가 좋은 청년. 오늘은 그래프 이론 중 하나인 위상 정렬을 사용하여 풀이하는 문제를 풀어 봤다.

> ### 위상 정렬 이란?

**위상 정렬(Topological Sort)** : '순서가 정해져 있는 작업'을 차례대로 수행해야 할 때, 그 **작업의 순서를 결정해 주는 알고리즘**이다.

> #### 위상 정렬의 필수 조건 : DAG
>
> 위상 정렬은 아무 그래프에서나 쓸 수 없고, 반드시 **DAG (Directed Acyclic Graph, 유향 비순환 그래프)** 구조여야 한다.

1. **방향성(Directed):** 선후 관계가 명확해야 한다. (A $\rightarrow$ B)
2. **비순환(Acyclic):** 사이클(순환)이 없어야 한다. 만약 "A $\rightarrow$ B $\rightarrow$ C $\rightarrow$ A" 처럼 꼬리를 물면 '닭이 먼저냐 달걀이 먼저냐'의 딜레마에 빠져 영원히 순서를 정할 수 없다.

> #### 동작 과정

1. 그래프의 모든 노드의 진입 차수를 계산
2. 진입 차수가 **0**인 노드를 큐에 전부 넣는다.
3. 큐에서 노드를 하나 꺼내어 결과 리스트에 추가(출력)
4. 꺼낸 노드와 연결된(뻗어 나가는) 간선을 모두 제거 (연결되어 있던 노드들의 진입 차수를 1씩 감소시킨다.)
5. 새롭게 진입 차수가 **0**이 된 노드가 있다면 큐에 넣는다.
6. 큐가 빌 때까지 3~5번 과정을 반복

<br>

### 시간 복잡도

위상 정렬의 시간 복잡도는 **$O(V + E)$**  
 ($V$: 노드의 수, $E$: 간선의 수)
각 노드를 큐에 한 번씩 넣고 빼며, 그 과정에서 모든 간선을 딱 한 번씩만 확인하기 때문에 그래프의 크기만큼만 시간이 소요되는 **매우 빠르고 효율적인 알고리즘**이다.

<br>

### 주로 사용되는 경우는??

위상 정렬은 단순히 코딩 테스트용 알고리즘이 아니라, 우리가 매일 사용하는 개발 도구와 시스템의 뼈대를 이루는 핵심 원리이다.

### 1. 패키지 의존성 관리 (NPM, Gradle, Pip)

우리가 `npm install`이나 `gradle build`를 실행할 때, 패키지 매니저는 수백 개의 라이브러리를 다운로드한다. 이때 라이브러리 간의 의존성(A를 깔려면 B가 먼저 필요함)을 분석하여 **가장 먼저 설치해야 하는 패키지부터 순서대로 설치**하게 해주는 것이 바로 위상 정렬

### 2. 빌드 시스템 및 CI/CD 파이프라인

- **빌드 도구 (Make, Webpack):** 여러 소스 코드 파일들이 서로를 참조(`import`)할 때, 어떤 파일부터 먼저 컴파일해서 묶어(Bundling) 나가야 할지 결정한다.
- **CI/CD (GitHub Actions, Jenkins):** "테스트가 끝나야 배포를 한다", "DB 세팅이 끝나야 백엔드를 띄운다"처럼 파이프라인의 작업 순서와 병렬 처리 가능 구간을 스케줄링할 때 사용된다.

### 3. 스프링(Spring) 프레임워크의 Bean 초기화

스프링과 같은 프레임워크에서 객체(Bean)들을 생성하고 의존성을 주입(DI)할 때, 참조하는 객체가 먼저 생성되어 있어야 한다. 스프링 컨테이너는 내부적으로 의존성 그래프를 그려 위상 정렬을 통해 생성 순서를 결정하며, 만약 '순환 참조(사이클)'가 발생하면 즉시 애플리케이션을 종료시켜 버린다.

### 4. 데이터베이스 테이블 생성 및 삭제 (FK 제약조건)

RDBMS에서 외래키(Foreign Key)로 묶인 수십 개의 테이블을 생성하거나 통째로 삭제(Drop)해야 할 때, 부모 테이블과 자식 테이블의 순서를 잘못 맞추면 에러가 발생한다. 이때 테이블 간의 참조 관계를 위상 정렬하여 쿼리 실행 순서를 결정한다.

> ### 게임 개발에서는 위상 정렬을 어떻게 사용할 수 있을까?

#### 1. 스킬 트리(Skill Tree) 및 테크 트리(Tech Tree) 검증

RPG 게임의 스킬 트리나 전략 시뮬레이션의 테크 트리는 DAG(Directed Acyclic Graph)구조이다.

- **활용** : 기획자가 수백 개의 스킬과 선행 조건을 엑셀이나 툴로 설계할 때, 실수로 "파이어볼을 배우려면 메테오가 필요하고, 메테오를 배우려면 파이어볼이 필요하다" 같은 식의 순환 논리 오류를 범할 수 있다. $\rightarrow$ 사이클 오류를 사전에 찾아내어 경고 해주고, 유저가 특정 스킬을 찍었을 때, '다음에 해금될 스킬 목록'을 순서대로 연산할 수도 있다.

#### 2.퀘스트 및 스토리 진행 시스템

메인 퀘스트와 수많은 서브 퀘스트가 존재하는 게임에서 활용되기도 한다.

- 특정 NPC가 죽는 퀘스트(A)가 진행되기 전에, 그 NPC와 대화해야 하는 서브 퀘스트(B)가 무조건 먼저 완료되어야 하는 등의 복잡한 선후 관계가 생긴다.
- 이 때, 위상정렬을 사용하여 현재 유저가 진행할 수 있는 퀘스트의 목록을 핉터링하여 유저에게 보여줄 수 있다.

#### 4. 게임 AI의 목표 지향적 행동 계획

스마트한 적 AI는 목표를 달성하기 위해 스스로 행동 순서를 계획한다.
GOAP(Goal-Oriented Action Planning)기법을 구현할 때 사용되기도 한다.
명작 FPS게임인 F.E.A.R에서 처음 도입되어 극찬을 받기도 했다.

- 기존의 AI는 "플레이어가 보이면 쏜다. 탄약이 없으면 재장전한다"식의 단순한 스크립트(if-else)로 움직였다. 하지만 상황이 복잡해지면 AI가 바보처럼 느껴지기 마련이다.
- **위상 정렬의 적용** : GOAP를 사용하는 AI는 목표(Goal)를 가진다.

GOAP는 충분히 훌륭한 기법이지만 결국 개발자가 미리 짜놓은 규칙에서 움직인다. 생성형 AI를 잘 사용하지 못하면 뒤쳐지는 현재 , 넥슨 자회사인 엠바크 스튜디오에서 출시한 아크레이더스에는 실제로 머신러닝을 통해 수년간 학습한 데이터로 움직이는 적대 몹이 있다. 머신러닝과 생성형 AI는 좀 다르지만 이렇게 머신러닝을 극한으로 적용한 사례를 보니 정말 놀라웠다. 개인이 구현하기에는 굉장히 난이도가 높은 기술이지만, 언젠가 한번쯤은 도전해보는 것도 나쁘지 않다고 생각한다.
