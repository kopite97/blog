### 문제 설명

n명의 권투선수가 권투 대회에 참여했고 각각 1번부터 n번까지 번호를 받았습니다. 권투 경기는 1대1 방식으로 진행이 되고, 만약 A 선수가 B 선수보다 실력이 좋다면 A 선수는 B 선수를 항상 이깁니다. 심판은 주어진 경기 결과를 가지고 선수들의 순위를 매기려 합니다. 하지만 몇몇 경기 결과를 분실하여 정확하게 순위를 매길 수 없습니다.

선수의 수 n, 경기 결과를 담은 2차원 배열 results가 매개변수로 주어질 때 정확하게 순위를 매길 수 있는 선수의 수를 return 하도록 solution 함수를 작성해주세요.

### 제한 사항

- 선수의 수는 1명 이상 100명 이하입니다.
- 경기 결과는 1개 이상 4,500개 이하입니다.
- results 배열 각 행 [A, B]는 A 선수가 B 선수를 이겼다는 의미입니다.
- 모든 경기 결과에는 모순이 없습니다.

---

### 풀이

```java
class Solution {
    public int solution(int n, int[][] results) {
        int answer = 0;

        int[][] graph = new int[n+1][n+1];

        for (int i = 0; i < results.length; i++) {

            int winner = results[i][0];
            int loser = results[i][1];

            graph[winner][loser] = 1;
            graph[loser][winner] = 2;
        }

        for (int k = 1; k <= n; k++) {
            for (int i = 1; i <= n; i++) {
                for (int j = 1; j <= n; j++) {
                    if (graph[i][k] == 1 && graph[k][j] == 1) {
                        graph[i][j] = 1;
                        graph[j][i] = 2;
                    }
                }
            }
        }

        for (int i = 1; i <= n; i++) {
            boolean isRank = true;
            for (int j = 1; j <= n; j++) {
                if(i == j ) continue;
                if (graph[i][j] != 1 && graph[i][j] != 2) {
                    isRank = false;
                    break;
                }
            }
            if (isRank) {
                answer++;
            }
        }


        return answer;
    }
}
```

플로이드-위셜 (Floyd-Warshall) 알고리즘을 활용하는 문제.  
프림,다익스트라,크루스칼에 이어 한번 다른 그래프 관련 알고리즘 이다.

크루스칼(간선 정렬 + 유니온 파인드)과 프림(우선순위 큐)은 최소 신장 트리(MST)를 구현하는데 사용되는 알고리즘이고, 다익스트라(1: 다수 최단경로 + 우선순위 큐)와 플로이드-위셜(다수 :다수 최단경로 + 3중 for)은 최단 경로를 구하는 데 사용하는 알고리즘이다.

#### 플로이드 위셜 (Floyd_Warshall)

- **다익스트라(Dijkstra):** 1대 다(1:N) 최단 경로
- **플로이드-워셜(Floyd-Warshall):** 다대 다(N:N) 최단 경로

<br>

## 핵심 원리: "거쳐 가는 환승역(K)을 찾아라!"

플로이드-워셜 알고리즘의 핵심 아이디어는 단 하나의 문장으로 요약할 수 있다.

> **"출발지(A)에서 도착지(B)로 다이렉트로 가는 것보다, 중간에 K를 거쳐서 가는 게 더 빠를까?"**

즉, 모든 노드를 한 번씩 `중간 기착지(K)`로 설정해 놓고, 기존에 알고 있던 최단 거리보다 `K`를 거쳐 가는 거리가 더 짧다면 그 값으로 계속 갱신해 나가는 방식이다.

이를 점화식으로 표현하면 다음과 같다.

> $D[i][j] = \min(D[i][j], D[i][k] + D[k][j])$

<br>

## Java 코드 구현 (3중 for문)

플로이드-워셜 알고리즘은 2차원 배열과 3중 `for`문만으로 매우 직관적이고 짧게 구현할 수 있다.

```java
public class FloydWarshall {
    static final int INF = 1000000000; // 무한대를 의미하는 값 (오버플로우 방지)

    public static void main(String[] args) {
        int n = 4; // 노드의 수
        int[][] graph = new int[n + 1][n + 1];

        // 1. 그래프 초기화: 자기 자신으로 가는 길은 0, 나머지는 무한대(INF)로 설정
        for (int i = 1; i <= n; i++) {
            for (int j = 1; j <= n; j++) {
                if (i == j) graph[i][j] = 0;
                else graph[i][j] = INF;
            }
        }

        // (여기서 간선 정보(비용)를 입력받아 graph 배열에 채워 넣는다.)

        // 2. 플로이드-워셜 알고리즘 핵심 로직
        // 거쳐가는 노드(k)가 반드시 가장 바깥쪽 반복문이어야 한다!!
        for (int k = 1; k <= n; k++) {          // 거쳐가는 노드
            for (int i = 1; i <= n; i++) {      // 출발 노드
                for (int j = 1; j <= n; j++) {  // 도착 노드
                    // i에서 j로 가는 기존 거리보다 k를 거쳐가는 거리가 짧다면 갱신
                    if (graph[i][k] + graph[k][j] < graph[i][j]) {
                        graph[i][j] = graph[i][k] + graph[k][j];
                    }
                }
            }
        }
    }
```
